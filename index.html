Solar System

I created a solar system with dynamic physics in OpenGL using C++. OpenGL is a graphics library that lets users create GPU programs called shaders, which are incredibly optimized to run on the GPU and are completely programable, making them very flexible and powerful.

Notable features:

  • Dynamic orbital physics with second order numerical integration
  • WASD, mouse, up, down, and roll movement
  • control over speed of ojbects
  • Blinn-Phong lighting
  • Shadow maps
  • Spotlights
  • Skybox
  • Models
  • Bloom

This project helped me attain a system-level understanding of the computer by working closely with GPU and needing to apply optimizations to maintain a reasonable framerate. I also needed to become very diligent in managing pointers to objects. I needed a very intimate understanding of the concepts and techniques of software design. This vast project required excellent organizational skills, and I needed to employ many fundamental programming concepts to make the project flow smoothly. An example is that I used inheritance extensively in this project since many classes share similar functionality, especially the suns, planets, and models. This project is probably the most I've ever used OOP concepts.

This project was entirely individual, requiring me to find my own resources to continue adding features and learning about OpenGL, which was a great experience and made me much more confident to tackle challenging projects like this in the future. Towards the end of the project, many features and optimizations I faced were niche, such as bloom optimizations, forcing me to find obscure resources that hardly fit my project to get by. This leads to the next learning goal I met: developing effective problem-solving skills. Problem-solving in shaders is exceptionally difficult, requiring me to get creative to find issues since there are no print statements in GLSL. Additionally, since I made this program in C++ and used pointers frequently, I encountered segmentation faults almost constantly, which are incredibly difficult to debug, and this gave me great experience solving complex problems.

Ray Tracer

This project was my first experience with computer graphics. I made this ray tracer using Python and minimal third-party libraries. Some notable features include support for multiple objects, smooth shading, triangle meshes, directional and point lights, Phong Lighting, shadows, and 3D transformations.

This project was a significant milestone in my computer science academic career because it was one of the most challenging projects I faced at the time. I was unfamiliar with the mathematics required for this project, which forced me to do my own extensive research to eventually get it working.

Goals: This project made great use of object-oriented programming and inheritance, which helped me understand the concepts and techniques of software design. This project was a massive undertaking for me then, which gave me experience working individually on a project. Since the challenges in this program were unique, this project made me better at problem-solving in general, which is great because I did this project in the first semester of my second year. Most of the issues I encountered in this project were visual anomalies with the images, requiring me to be creative with how I would find more information about these issues.

Minijava Compiler

The Minijava compiler mimics the behavior of the Java compiler, but all the logic is built from scratch. The primary technology I used was ANTLR, and some of the key features include:

  • Class and block structure (flow control)
  • JVM assembly code generation
  • Lexing and parsing
  • Package inclusion
  • Print statements
  • AST generation
  • Type checking
  • Method calls

Goals: This project primarily helped me achieve a greater system-level understanding of the computer. This was one of the most complex projects I've ever done, and there were several weeks where my team and I would get together and spend more than 20 hours on it in a week. This gave me significant experience working with others. This project also generated some of the most mind-boggling errors I've ever encountered, and this project is mainly responsible for most of my current debugging/problem-solving skills.
